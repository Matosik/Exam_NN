# Тренажёр НСиГО

Приложение‑тренажёр для подготовки к тесту по курсу **«Нейронные сети и глубокое
обучение»**. Карточки с 4 вариантами ответа, переворот с подсветкой (зелёный —
верно, красный — неверно), правильный ответ и краткое пояснение. Есть выбор числа
вопросов, итоговая сводка с разбором ответов, интервальное повторение и экран
прогресса.

База вопросов: **254 вопроса** по 6 темам, сгенерированы по материалам курса
(структура лекций, транскрипции, методички по ИНС/СНС/РНС/трансформерам).

Рядом есть страница‑**справочник** (`reference.html`): ключевые формулы, факты
«100% знать» и глоссарий терминов, сгруппированные по темам, с поиском. Открывается
из тренажёра по ссылке «Справочник» в шапке.

## Как запустить

**Простой способ — двойной клик по `index.html`.** Вопросы подхватываются из
`data/questions.js`, поэтому всё работает офлайн без сервера.

**Через локальный сервер** (если хотите грузить данные из `data/questions.json`
напрямую через `fetch`):

```bash
python serve.py
```

Откроется `http://localhost:8000/index.html`.

> Почему два файла с данными? Браузеры блокируют `fetch()` локального JSON при
> открытии страницы по `file://`. Поэтому источником истины остаётся
> `data/questions.json`, а `data/questions.js` — это его автогенерируемая копия
> (одна строка `window.QUIZ_DATA = …`), нужная для запуска двойным кликом.

## Как редактировать вопросы

1. Откройте `data/questions.json`. Формат одного вопроса:

   ```json
   {
     "id": "t4-012",
     "topic": "t4",
     "q": "Текст вопроса?",
     "a": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
     "correct": 0,
     "explanation": "Краткое пояснение."
   }
   ```

   - `topic` — ключ темы из секции `topics` (`t1`…`t6`).
   - `correct` — индекс правильного варианта в массиве `a` (0–3). В приложении
     порядок вариантов перемешивается.
   - `id` — стабильный идентификатор; по нему хранится прогресс, не меняйте его у
     существующих вопросов.

2. Пересоберите бандл:

   ```bash
   python tools/build.py
   ```

   Скрипт проверит данные (ровно 4 варианта, корректный `correct`, уникальные `id`)
   и обновит `data/questions.js` **и** `data/reference.js`.

## Как редактировать справочник

Содержимое страницы `reference.html` лежит в `data/reference.json`. Каждая тема —
объект с массивами `formulas` (`{name, expr, note}`), `facts` (строки) и `glossary`
(`{term, def}`). После правок запустите `python tools/build.py`, чтобы обновить
`data/reference.js`.

## Как работает интервальное повторение

Для каждого вопроса хранится «ящик» Лейтнера (0–5):

- правильный ответ поднимает вопрос на ящик выше → показывается **реже**;
- ошибка опускает на два ящика ниже → показывается **чаще**;
- ящик ≥ 4 — статус **«выучен»** (такие вопросы почти не попадаются).

Вопросы для сессии выбираются случайно, но с весом по ящику: трудные —
значительно чаще лёгких. Весь прогресс сохраняется в `localStorage` браузера и
виден на вкладке **«Прогресс»**. Там же можно сбросить статистику.
